# Бот с Марковской цепью для режима обучения

## Архитектура

### Структура файлов

```
server/src/
├── domain/
│   └── bot/
│       ├── handStrengthEvaluator.ts  # Оценка силы руки (WEAK/MEDIUM/STRONG)
│       ├── markovModel.ts            # Марковская цепь с матрицей переходов
│       └── markovBot.ts              # Основной класс бота
└── application/
    └── trainingGameService.ts        # Сервис для интеграции бота в игру
```

## Компоненты

### 1. handStrengthEvaluator.ts
Оценивает силу руки игрока на основе:
- Карт игрока (2 карты)
- Общих карт на столе
- Фазы игры (PRE_FLOP, FLOP, TURN, RIVER)

Возвращает категорию: `WEAK`, `MEDIUM` или `STRONG`.

### 2. markovModel.ts
Реализует Марковскую цепь для принятия решений:
- **Состояние**: фаза игры + сила руки + последнее действие оппонента
- **Действия**: FOLD, CHECK, CALL, RAISE, ALL_IN
- **Матрица переходов**: вероятности выбора действий для каждого состояния
- **Обучение**: обновление весов после каждого раунда

### 3. markovBot.ts
Основной класс бота:
- Получает решение от MarkovModel
- Преобразует решение в игровое действие с учетом ограничений
- Сохраняет историю для обучения

### 4. trainingGameService.ts
Сервис для интеграции:
- Управляет жизненным циклом бота
- Выполняет ходы бота с задержкой (1-2 сек)
- Обновляет модель после раундов

## Использование

Бот автоматически активируется в режиме обучения (`isTraining: true`).

### Уровни сложности

```typescript
enum BotDifficulty {
  EASY,    // Консервативная игра
  NORMAL,  // Сбалансированная игра
  HARD     // Агрессивная игра
}
```

### Логирование

Все решения бота логируются в консоль:
```
Bot decision: {
  state: { phase, handStrength, opponentLastAction },
  action: BotAction,
  decision: { action, amount },
  botChips, currentBet, pot
}
```

## Интеграция с FSM

Бот полностью интегрирован в существующий FSM:
- Использует те же состояния (PRE_FLOP, FLOP, TURN, RIVER)
- Применяет действия через `applyPlayerAction`
- Обновляется после каждого раунда (WIN/LOSS)

## Математика Марковской модели

**Вероятность действия**: `P(action | state) = transitionMatrix[stateKey][action]`

**Ключ состояния**: `${phase}_${handStrength}_${opponentLastAction}`

**Выбор действия**: Weighted random selection по кумулятивному распределению
